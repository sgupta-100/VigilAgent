Generated PDF reports from security scans and penetration tests.

```
reports/
├── uuid-reports/        # Reports with UUID filenames (100+ files)
```

## 📄 Report Categories

### UUID Reports (`uuid-reports/`)
**Format:** `[uuid].pdf`  
**Example:** `01ce448e-c80f-4eba-afa5-59da50ac53a3.pdf`

Reports generated with UUID-based scan IDs. These are production scan reports.

**Count:** ~100 files

---

### Named Reports (`named-reports/`)
**Format:** `Scan_Report_[name].pdf`  
**Examples:**
- `Scan_Report_1.pdf` through `Scan_Report_19.pdf`
- `Scan_Report_AUDIT_1772472824.pdf`
- `Scan_Report_Consolidated_Intelligence.pdf`
- `Scan_Report_EMPTY_TEST.pdf`
- `Scan_Report_MANUAL_DEBUG_TEST.pdf`
- `Scan_Report_PRO-VERIFICATION-777.pdf`
- `Scan_Report_TEST_1772472026.pdf`
- `Scan_Report_VERIFICATION_TEST_001.pdf`

Reports with descriptive names for specific test scenarios.

**Count:** ~100 files

---

### Torture Tests (`torture-tests/`)
**Format:** `Scan_Report_TORTURE-P[phase]-[id].pdf`  
**Examples:**
- `Scan_Report_TORTURE-P1-24465296.pdf`
- `Scan_Report_TORTURE-P20-CANCEL-aae5c99a.pdf`

Stress test reports from torture testing phases (P1-P20).

**Phases:**
- P1-P3: Initial torture tests
- P5-P20: Progressive stress testing
- CANCEL: Cancelled torture tests

**Count:** ~200 files

---

### Benchmark Tests (`benchmark-tests/`)
**Format:** `Scan_Report_EXTREME-[type]-V6-[number].pdf`  
**Examples:**
- `Scan_Report_EXTREME-BENCHMARK-V6-1.pdf` through `V6-10.pdf`
- `Scan_Report_EXTREME-STRESS-V6-1.pdf` through `V6-10.pdf`
- `Scan_Report_EXTREME-STRESS-FINALE-1.pdf`

Extreme benchmark and stress test reports.

**Types:**
- BENCHMARK: Performance benchmarks
- STRESS: Stress testing
- FINALE: Final stress tests

**Count:** ~20 files

## 📊 Report Format

All reports are PDF files generated by the Vigilagent reporting system.

### Report Contents
- **Executive Summary** - High-level findings
- **Scan Metadata** - Target, duration, configuration
- **Vulnerability Details** - Discovered vulnerabilities with evidence
- **Attack Timeline** - Chronological attack sequence
- **Recommendations** - Remediation guidance
- **Technical Details** - Full technical analysis

### Report Generation
Reports are generated automatically after scan completion using:
- `backend/core/reporting.py` - Report generation engine
- `backend/reporting/` - Report templates and formatters

## 🔍 Finding Reports

### By Scan ID
```bash
# Find report by UUID
ls reports/uuid-reports/[scan-id].pdf

# Find report by name
ls reports/named-reports/Scan_Report_[name].pdf
```

### By Test Type
```bash
# Torture tests
```

### Search All Reports
```bash
# Search by pattern
dir reports\*\*TORTURE*.pdf

# Count reports
(dir reports\*\*.pdf -Recurse).Count
```

## 🗄️ Report Management

### Archiving Strategy
- **Active Reports:** Keep last 30 days
- **Archive:** Move reports older than 30 days to archive
- **Cleanup:** Delete reports older than 90 days

### Backup
- Reports should be backed up regularly
- Consider cloud storage for long-term retention
- Compress old reports to save space

### Disk Space
Monitor disk usage:
```bash
# Check reports directory size
du -sh reports/

# Find largest reports
du -h reports/*/*.pdf | sort -rh | head -20
```

## 🔒 Security Considerations

### Sensitive Information
Reports may contain:
- Target URLs and IP addresses
- Discovered vulnerabilities
- Exploit evidence (screenshots, payloads)
- Network traffic captures

### Best Practices
1. **Restrict Access:** Limit read permissions to authorized users
2. **Encrypt Storage:** Consider encrypting reports at rest
3. **Secure Transfer:** Use encrypted channels for report sharing
4. **Redaction:** Redact sensitive info before sharing externally
5. **Retention Policy:** Follow data retention policies
